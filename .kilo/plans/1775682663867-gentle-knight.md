# ISSUE: "Proceed to Payment" Button Not Working

## Problem Identified
User reports that the "Proceed to Payment" button in `new_group.ejs` is not working. When clicked, it doesn't advance to Step 2 (payment method selection).

## Analysis of `proceedToPayment()` Function (lines 1009-1021)

```javascript
function proceedToPayment() {
  totalMembersCount = parseInt(document.getElementById('totalMembers').value);
  if (isNaN(totalMembersCount) || totalMembersCount < MIN_MEMBERS || totalMembersCount > MAX_MEMBERS) {
    showNotification(`Membership count must be between ${MIN_MEMBERS} and ${MAX_MEMBERS}.`);
    return;
  }
  totalPayableAmount = totalMembersCount * BASE_FEE;
  document.getElementById('countDisplay').textContent = totalMembersCount;
  document.getElementById('totalFeeDisplay').textContent = `KES ${totalPayableAmount.toLocaleString()}.00`;
  document.getElementById('feeRateDisplay').textContent = `KES ${BASE_FEE.toLocaleString()} per member`;
  document.getElementById('stkAmountDisplay').textContent = `KES ${totalPayableAmount.toLocaleString()}.00`;
  showStep('step2');
}
```

## Potential Issues

### 1. **Input Validation Fails**
- `totalMembers` input value is empty, NaN, or out of range
- `MIN_MEMBERS` = 10, `MAX_MEMBERS` = 40 (from tbank.json compliance)
- If user enters < 10 or > 40, validation fails and shows error

### 2. **DOM Element Not Found**
- `getElementById('totalMembers')` returns null
- Display elements (`countDisplay`, `totalFeeDisplay`, etc.) not found

### 3. **showStep Function Issue**
- `showStep('step2')` doesn't work if step2 div doesn't exist or CSS is wrong

### 4. **JavaScript Errors**
- Syntax error or undefined variables preventing execution
- Console errors blocking function execution
